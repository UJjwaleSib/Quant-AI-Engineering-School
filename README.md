# AIEngSchool — Quant + AI Engineering Learning Platform

A personal, AI-powered learning system that teaches quant research and AI engineering by **building real artifacts**, not passive reading.

## Architecture

```
├── api/                    # FastAPI backend (Python)
│   ├── main.py             # App entry point
│   ├── models.py           # SQLAlchemy ORM models
│   ├── schemas.py          # Pydantic request/response schemas
│   ├── database.py         # Async SQLite + settings
│   ├── routers/            # auth, lessons, exercises, research_logs, progress, chat
│   ├── services/llm.py     # Claude API integration
│   └── content/curriculum.py  # All 16 modules + lessons + exercises
│
└── aiengschool-web/        # Next.js 14 frontend
    ├── app/
    │   ├── dashboard/      # Progress overview
    │   ├── modules/        # Curriculum browser + lesson viewer
    │   ├── playground/     # Live Python editor (Piston API)
    │   └── research-logs/  # Auto-generated research memos
    └── components/
        ├── ExerciseEngine  # Monaco editor + Piston + AI feedback
        ├── AIChat          # Floating Claude tutor
        ├── ResearchLogCard # Expandable log viewer
        └── Sidebar/ModuleCard/ProgressRing
```

## Curriculum

**Track 1 — Quant Research Foundations**
- Q1: Python for Quant Research (NumPy, Pandas, yfinance)
- Q2: Backtesting & Strategy Evaluation
- Q3: Statistical Thinking for Quants
- Q4: Market Regime Modeling
- Q5: Portfolio Construction
- Q6: Research Workflow & Memo Writing

**Track 2 — AI Engineering Systems**
- A1–A3: Foundations (Python, APIs, LLM Integration)
- A4–A5: Backend Systems (FastAPI, Databases)
- A6–A7: AI Systems (Prompt Chains, Tool Use, RAG)
- A8–A9: Speed + Real-World (AI-Assisted Dev, Async)
- A10: Capstone — End-to-End AI + Fintech System

## Quick Start (Local)

### Backend

```bash
cd api
cp .env.example .env
# Add your ANTHROPIC_API_KEY to .env

pip install -r requirements.txt
uvicorn api.main:app --reload --port 8000
# API docs: http://localhost:8000/docs
```

### Frontend

```bash
cd aiengschool-web
cp .env.local.example .env.local
# NEXT_PUBLIC_API_URL=http://localhost:8000

npm install
npm run dev
# App: http://localhost:3001
```

### First Login

1. Open http://localhost:3001
2. Register with any email + password (stored locally in SQLite)
3. Start with Module Q1, Lesson 1

## Key Features

| Feature | How it works |
|---|---|
| **AI Tutor** | Claude Sonnet via Anthropic API, context-aware (knows current module + code) |
| **Exercise Engine** | Monaco editor → Piston API (free Python sandbox) → Claude feedback |
| **Research Logs** | Auto-generated by Claude every time you pass an exercise |
| **Progress Tracking** | SQLite, per-lesson completion, XP system, streak counter |
| **Playground** | Full Python REPL with 7 built-in quant snippets |

## Deploy

### Backend → Railway

```bash
# Install Railway CLI
npm install -g @railway/cli
railway login
railway init
railway up
# Set env vars in Railway dashboard: ANTHROPIC_API_KEY, SECRET_KEY
```

### Frontend → Vercel

```bash
npm install -g vercel
cd aiengschool-web
vercel
# Set NEXT_PUBLIC_API_URL to your Railway backend URL
```

## Design Philosophy

> Most people build a "course platform." This is a **quant + AI operator system.**

Every module produces **artifacts**:
- Exercises produce **Research Logs** (auto-generated by AI)
- Playground experiments produce **runnable code**
- Modules build toward a **deployable capstone system**

No passive learning. Every lesson ends with a real exercise. Every exercise produces a research log that compounds over time.
